# Security Policy

## Supported Versions

Currently, only the latest release of the Bento Guard SDK is fully supported with security updates. We strongly recommend always upgrading to the latest version to ensure your AI Agents are protected by the newest threat detection capabilities.

| Version | Supported          |
| ------- | ------------------ |
| >= 1.2.9 | :white_check_mark: |
| < 1.2.9  | :x:                |

## Reporting a Vulnerability

We take the security of the Bento Guard SDK and the assets it protects very seriously. If you believe you have found a security vulnerability in the SDK or the broader Bento architecture, please report it to us immediately.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them by using GitHub Security Advisories or sending an email to our security team.

### How to Report
1. **GitHub Security Advisory:** Navigate to the "Security" tab in this repository, click "Advisories", and use the "Report a vulnerability" button.
2. **Email:** You can alternatively report via email to: `security@bentoguard.xyz`

### What to Include
Please provide a detailed description of the issue, including:
- The type of vulnerability (e.g., cryptographic bypass, payload tampering, XSS, etc.).
- Steps to reproduce the issue.
- A proof-of-concept (PoC) if available.
- Any potential impact on users or AI Agents.

### What to Expect
- You will receive a response acknowledging receipt of your report within 48 hours.
- We will keep you updated on the progress of our investigation and the timeline for a fix.
- We will credit you for the discovery (if you desire) once the vulnerability has been resolved and disclosed.

Thank you for helping keep the Bento ecosystem safe!
